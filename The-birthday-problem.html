Problem: How many people need to be in a room in order for there to be a greater than 1/2 probability that at least two of them have the same birthday?

First lets start by defining some helpful functions

#generates random birthdays for n people 
birthday = function (n){
  sample(1:365, size = n, replace = T)
}

#checks if there is at least one matching birthday in a given array 
ismatch = function (n){
  for(i in (1:(length(n) - 1))){
    #print(i)
    for(j in (i + 1):length(n)){
      #print(j)
      if(n[i] == n[j]){
        return(TRUE)
      }
    }
  }
  return(FALSE)
}

Our Theoretical Solution \(P_{n} = 1 - (\frac{364}{365})^{\binom{n}{2}}\)

#caculate the theoretical probability for a room of k people
tprob = function(k){
  p = 1 - ((364/365)^(choose(k,2)))
  return(p)
}

Morrin’s Theoretical Solution \(P_{n} = 1-\Pi^{n-1}_{n=1}\frac{365-n}{365}\)

Let’s code Morrin’s Theoretical Model

morrinModel = function(n){
  if(n == 0 || n == 1)
    return(0)
  not = 1
  for(i in 1:(n-1)){
    not = not * ((365-i)/365)  
  }
  return (1 - not)
}

Lets run a simulation to check our theoretical probability formula for different numbers of people

#loop to simulate
loopControl = 2

#my model 
theoryP = vector()
theoryP[1] = 0

#morrin 
morrinP = vector()
morrinP[1] = 0

#experimental
prob = vector()
prob[1] = 0


while (loopControl <= 100)
{
  theoryP[loopControl] = tprob(loopControl)
  morrinP[loopControl] = morrinModel(loopControl)
  
  countFalse = 0 
  countTrue = 0
  for(i in 1:10000){
    x = birthday(loopControl)
    y = ismatch(x)
  
    if(y == TRUE)
      countTrue = countTrue + 1
    if(y == FALSE)
      countFalse = countFalse + 1
  }

  prob[loopControl] = countTrue/(countTrue + countFalse)
  
  loopControl = loopControl + 1
}

Conclusion: Although our model shares a similar error profile as Morin’s model at small values of n, Morin’s model is clearly a better theoretical solution to the problem statement. Furthermore, although our model was able to correct answer posed question, it is extremely ineffective at predicting the exact probablity. More specific for medium values of n, say 25-75, our model consistently underestimates the probability of finding at least one pair. Based on a two-sided t test, it is extreme unlikely that proability our model predicts for a room of 23 people is correct (p < 2.2e -16).

numPairs = function(n){
  count = 0
  for(i in 1:(length(n) - 1)){
    for(j in (i + 1):length(n)){
      if(n[i] == n[j])
        count = count + 1 
    }
  }
  return(count)
}
